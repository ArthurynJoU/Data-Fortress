## Project Description

**Data Fortress** is an educational serious game designed to interactively teach modern cybersecurity concepts. Built as a 3D Tower Defense game, it challenges players to protect a core server from various types of cyber threats by strategically deploying defensive network modules.

Instead of traditional fantasy elements, the game translates real-world networking and security terminology into engaging gameplay mechanics. 

### Key Features
* **Enemies:** Each enemy represents a specific cyber threat with unique behavioral patterns:
  * **DDoS:** Accelerates as it sustains damage.
  * **Ransomware:** Accumulates "anger" and periodically freezes nearby defenses to drain their health.
  * **Trojan:** Moves stealthily and remains untargetable until exposed by analytical effects.
  * **Worm:** A self-replicating threat that splits into smaller instances upon taking damage.
  * **Supply Chain Attack:** Bypasses local combat to inflict simultaneous global damage to active towers.
  * **Zero Day Exploit:** Periodically halts to execute a severe local attack, disabling nearby towers.
* **Defensive Modules (Towers & Effects):** Players utilize tools like *Firewall Gates*, *Antivirus Scanners*, *Patch Managers*, and *WAFs*, alongside effects like *IDS/DPI* and *Sandboxing* to control the flow of traffic.
* **Dynamic Routing:** Built on a custom BFS pathfinding algorithm that dynamically updates threat routes based on player tower placement.

## Technology Stack

* **Game Engine:** Unity 3D
* **Language:** C#
* **Architecture:** Object Pooling (`GameContentFactory`) for optimized memory management.
  * Custom State Machines for enemy spawning algorithms.
  * Event-driven UI updates (Observer pattern) via `GameSessionController` and `CardManager`.

## Getting Started

### Prerequisites
* Unity Hub installed on your machine.
* Unity Editor version **6000.2.9f1** (Unity 6).
* Visual Studio 2022 or any preferred C# IDE.
* Blender **5.1.2**.
* Canva.

### Installation & Run
1. Clone the repository to your local machine:
   ```bash
   git clone https://github.com/ArthurynJoU/Data-Fortress.git
   ```

2. Open Unity Hub, click **Open**, and select the cloned ``data-fortress`` folder.

3. Allow Unity to import all assets and resolve dependencies.

4. Navigate to ``Assets/Scenes/Levels`` and open the ``Menu``.

5. Press the **Play button** in the Unity Editor to start the simulation.

## Repository Structure

The project assets and source code are organised into the following directory structure:

* `Assets/Scripts/` — The core software architecture and game logic, divided into modular components:
    * `/Core/` — Base classes (`GameEntity`, `EnemyBase`, `TowerBase`, `EffectBase`), ScriptableObject configurations, and fundamental mathematical extensions.
    * `/Entities/` — Concrete implementations of game objects (Effects, Enemies, Towers, and Enums) used to drive prefab behavior.
    * `/Interfaces/` — Explicit code contracts (`IDamageable`, `ITickable`, etc.) decoupling system interactions.
    * `/Managers/` — Higher-level systems orchestrating state, network economy, and session loops.
    * `/UI/` — Graphical canvas controllers, tooltips, audio management, and main menu state machines.
* `Assets/Fonts/` — Contains the **Saira Stencil** variable and static typography assets used for the visual identity of the interface.
* `Assets/Models/` — Raw 3D meshes and Blender source files before asset processing and material assignment.
* `Assets/Prefabs/` — Fully configured, ready-to-spawn game objects (Towers, Threats, Levels, and structural grid boundaries).
* `Assets/Scenes/` — The primary deployment environments (`MainMenu` and `GameScene`).
* `Assets/ScriptableObjects/` — Data containers used for modifying game balance and level data without changing the underlying code.
* `Assets/Setup/` — Factory configurations managing object pooling allocation.
* `Assets/Sounds/` — Audio samples, environmental tracks, and sound effect libraries.
* `Assets/Videos/` — Media assets integrated into the interactive gameplay rules interface.
